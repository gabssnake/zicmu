# ingest/

Tools for adding new media to zicmu. Generates `albums.json` from raw album metadata, MusicBrainz track data, and cover art fetched from the Cover Art Archive.

## Prerequisites

- Node.js v24+
- `ffmpeg` in PATH (for silence detection during track-boundary analysis)
- Internet access (MusicBrainz and Cover Art Archive APIs, no auth needed)

## Pipeline overview

```
ingest/cache/albums-raw.json          ← edit this to add/remove albums
       │
       ├─ fetch-tracks.js     → ingest/cache/tracks/<id>.json  (track timestamps)
       ├─ fetch-covers.js     → media/<id>.jpg          (cover art)
       │
       └─ build-albums.js     → albums.json             (final output)
```

Run in order:

```bash
node ingest/fetch-tracks.js   # ~2 min per album (MusicBrainz rate limit)
node ingest/fetch-covers.js   # ~1 min per album
node ingest/build-albums.js   # fast, just assembles
```

## Adding a new album

1. Copy the MP3 file into `media/`
2. Add an entry to `ingest/cache/albums-raw.json` — see `ingest/add-album.md` for the format
3. Run the pipeline above
4. Check `node ingest/validate-reviews.js` — if the album is flagged, timestamps need manual review

## Reviewing timestamps

20 of 24 albums currently have `_review: true` because silence detection couldn't confirm all
track boundaries. Run:

```bash
node ingest/validate-reviews.js
```

This lists flagged albums and their track counts. See `ingest/verify-timestamps.md` for the
review workflow.

## Cache files

```
ingest/cache/
  albums-raw.json      Source of truth for album metadata (id, title, artist, year, filename)
  covers-report.json   Generated by fetch-covers.js — tracks cover fetch status per album
  tracks/              Generated by fetch-tracks.js — one JSON per album with track timestamps
```

`albums-raw.json` is the only file you should edit by hand. Everything else is generated and can be deleted and rebuilt.

## AI prompts

`ingest/add-album.md` — paste into an AI assistant to generate a new `ingest/cache/albums-raw.json` entry
`ingest/verify-timestamps.md` — guidance for manually reviewing and correcting track boundaries
